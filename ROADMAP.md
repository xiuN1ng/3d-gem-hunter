# 宝石猎人 · 项目路线图 (Roadmap)

> 版本：v0.1  |  更新时间：2026-07-14  |  状态：初版已提交

以 **3D 翡翠展示效果** 为核心，按以下优先顺序推进：

**优先顺序**：`2 视觉/材质` → `4 技术优化` → `3 UI/UX` → `1 玩法扩展` → `5 内容丰富`

---

## 阶段 0：基础稳固（已完成）

- 当前可玩的开石核心循环
- 程序化原石 + 单刀切割 + 估价交易
- Three.js + Vite MVP 已线上

---

## 阶段 1：视觉/材质升级（优先级 2）

**目标**：让翡翠的 3D 展示效果达到惊艳级别，成为游戏最强卖点。

### 关键里程碑
- [ ] 内部结构升级（体素 / SDF 数据层）
- [ ] 真实玉肉透光、折射、辉光
- [ ] Reveal Mask + 动态切面/裂纹表现
- [ ] 光照与材质系统优化（宝石级渲染）
- [ ] 多视角高保真展示（放大镜、灯光实验室等）

---

## 阶段 2：技术优化（优先级 4）

**目标**：代码健壮、可维护、高性能，为后续大规模内容做好准备。

### 关键里程碑
- [ ] 性能优化（帧率、内存、移动端）
- [ ] 代码拆分与模块化
- [ ] 测试覆盖率提升
- [ ] 构建/部署流程完善
- [ ] 资源加载策略

---

## 阶段 3：UI/UX 迭代（优先级 3）

**目标**：打造沉浸式、专业级的操作与展示体验。

### 关键里程碑
- [ ] 更沉浸的控制台界面
- [ ] 手机/平板适配
- [ ] 操作引导与教程
- [ ] 结果动画与反馈系统
- [ ] 整体视觉风格统一

---

## 阶段 4：玩法扩展（优先级 1）

**目标**：丰富核心玩法，增加深度与重玩性。

### 关键里程碑
- [ ] 多刀切割系统
- [ ] 风险与策略系统
- [ ] 赌石机制
- [ ] 成就/收藏/升级系统
- [ ] 进阶玩法循环

---

## 阶段 5：内容丰富（优先级 5）

**目标**：海量可玩内容与世界观。

### 关键里程碑
- [ ] 更多场口、皮壳、玉种类型
- [ ] 故事叙事与背景设定
- [ ] 收藏册、图鉴系统
- [ ] 季节/事件内容
- [ ] 社区/分享功能

---

## 并行开发说明

| 阶段 | 可并行内容 | 依赖关系 |
|------|-----------|----------|
| 阶段 1（视觉） | SDF/体素数据结构 + 材质与光照系统 + Reveal Mask | 相互依赖较强，建议小范围并行 |
| 阶段 2（技术） | 性能优化 + 代码模块化 + 测试/CI | 阶段 1 视觉核心稳定后可并行 |
| 阶段 3（UI/UX） | 响应式布局 + 新 UI 组件 + 操作引导 | 可与阶段 2 部分并行 |
| 阶段 4（玩法） | 多刀切割 + 风险系统 + 成就框架 | 强依赖前三阶段 |
| 阶段 5（内容） | 数据表扩展 + 故事与图鉴 | 可从阶段 1 后期开始准备 |

**并行原则**：
1. 视觉基础（阶段 1）优先保证
2. 技术优化（阶段 2）可与视觉后期、UI 早期并行
3. 内容生产（阶段 5）可提前准备数据与美术资源
