# 宝石猎人 · 项目路线图 (Roadmap)

> 版本：v0.4  |  更新时间：2026-07-16  |  状态：阶段2技术优化已完成，阶段1视觉/材质升级进行中（更多细节同步），阶段3 UI/UX 并行推进

以 **3D 翡翠展示效果** 为核心，按以下优先顺序推进：

**优先顺序**：`2 视觉/材质` → `4 技术优化` → `3 UI/UX` → `1 玩法扩展` → `5 内容丰富`

---

## 阶段 0：基础稳固（已完成）

- 当前可玩的开石核心循环
- 程序化原石 + 单刀切割 + 估价交易
- Three.js + Vite MVP 已线上

---

## 阶段 1：视觉/材质升级（优先级 2）

**目标**：让翡翠的 3D 展示效果达到惊艳级别，成为游戏最强卖点。

### 关键里程碑
- [x] 内部结构升级（体素 / SDF 数据层）
  - 程序化 JadeVolume 数据结构
  - 支持密度、水头、颜色、棉絮、裂纹多属性采样
  - 确定性噪声系统（mulberry32 + fbm + valueNoise）
- [ ] 真实玉肉透光、折射、辉光
  - 体积透射与次表面散射参数调优
  - 折射率与IOR 模拟（类似宝石物理）
  - 动态辉光与自发光增强
- [ ] Reveal Mask + 动态切面/裂纹表现
  - 切面与玉肉融合的 Reveal Mask
  - 动态裂纹线绘制与传播效果
  - 切割时实时裂纹可视化
- [ ] 光照与材质系统优化（宝石级渲染）
  - MeshPhysicalMaterial 宝石级参数（transmission、thickness、ior、attenuation）
  - 环境光 + 点光 + 聚光多光源宝石渲染
  - 实时光照与材质响应
- [ ] 多视角高保真展示（放大镜、灯光实验室等）
  - 放大镜/聚焦相机系统
  - 灯光实验室模式（多角度光照测试）
  - 高分辨率截图与分享导出

### 当前推进：SDF 与材质视觉系统
- [x] JadeVolume + 程序化噪声模块已落地，支持密度、水、色、棉、裂纹采样
- [x] 切面纹理通过体积采样生成（512px），支持裂纹线绘制
- [x] Web Worker 迁移切面 SDF 采样，避免主线程阻塞
- [x] 基础体积数据结构与采样已集成到切割流程
- [ ] 进一步优化透射、折射与辉光参数调优（MeshPhysicalMaterial 参数精细化）
- [ ] Reveal Mask 实现动态切面/裂纹与玉肉融合
- [ ] 宝石级光照系统与多光源渲染优化
- [ ] 多视角高保真工具（放大镜、灯光实验室、导出功能）

**阶段1视觉目标**：从当前 SDF 基础出发，快速实现真实玉肉视觉冲击力，成为项目核心卖点。

## 阶段 2：技术优化（优先级 4）

**目标**：代码健壮、可维护、高性能，为后续大规模内容做好准备。

### 关键里程碑
- [x] 性能优化（帧率、内存、移动端）
- [x] 代码拆分与模块化
- [x] 测试覆盖率提升
- [x] 构建/部署流程完善
- [x] 资源加载策略

### 当前推进：性能优化（已完成）

- [x] 切面 SDF 采样迁移到 Web Worker，避免切割时阻塞主线程
- [x] 几何体、皮壳纹理与切面纹理复用，减少 GPU 重复上传和内存抖动
- [x] 移动端分档：低细分原石、128px 皮壳纹理、精简粒子与透射效果
- [x] Three.js 运行时与业务入口分包，SDF 同步回退按失败场景动态加载
- [x] 自动化性能预算纳入 CI 与部署流水线
- [x] 移动端 WebGL 自动验收：完整切割、FPS/P95、长任务与 GPU 资源预算
- [x] 发布后真机观测框架已准备（性能诊断模块 + 策略模块）

**阶段2总结**：核心性能瓶颈已解决，代码结构清晰，测试与CI完善。后续维护性优化可并行。

## 阶段 3：UI/UX 迭代（优先级 3）

**目标**：打造沉浸式、专业级的操作与展示体验。

### 关键里程碑
- [ ] 更沉浸的控制台界面
  - 东方机械风仪表盘（温度、进度、估价实时反馈）
  - 滑块/旋钮交互优化（平滑动画、触感反馈）
  - 结果卡片增强（宝石详情、价值曲线图、历史对比）
- [ ] 手机/平板适配
  - 响应式布局与断点优化
  - 触摸手势支持（旋转、缩放、滑动切割、双指操作）
  - 移动端性能与UI简化（精简控件、触控友好）
- [ ] 操作引导与教程
  - 新手引导流程（分步骤高亮 + 交互式演示）
  - 工具提示、帮助面板与上下文说明
  - 进阶技巧提示系统
- [ ] 结果动画与反馈系统
  - 切割过程增强动画（粒子、辉光、相机聚焦）
  - 结果揭晓特效（揭幕、价值弹出、风险提示）
  - 即时视觉/听觉反馈（成功/失败/风险动画）
- [ ] 整体视觉风格统一
  - 主题色彩、字体与图标系统
  - 一致的阴影、边框、过渡动画曲线
  - 暗黑/机械风与宝石辉光协调

### 当前推进：UI/UX 基础与响应式（并行启动）
- [x] 现有控制台、滑块、结果卡片、温度/进度UI 基础实现（main.js + style.css）
- [ ] 东方机械风格控制台细化（仪表盘增强、滑块触感、结果卡片升级）
- [ ] 移动端触摸优化与布局适配
- [ ] 新手引导流程设计与实现
- [ ] 结果动画与即时反馈系统规划
- [ ] 视觉风格统一与主题系统

**阶段3并行原则**：与阶段1视觉后期并行推进，优先响应式与控制台沉浸感，为后续玩法扩展打好交互基础。

## 并行开发说明

| 阶段 | 可并行内容 | 依赖关系 |
|------|-----------|----------|
| 阶段 1（视觉） | SDF/体素数据结构 + 材质与光照系统 + Reveal Mask | 相互依赖较强，建议小范围并行；当前SDF基础已稳 |
| 阶段 2（技术） | 性能优化 + 代码模块化 + 测试/CI | 已完成，可与阶段1后期、阶段3并行 |
| 阶段 3（UI/UX） | 响应式布局 + 新 UI 组件 + 操作引导 + 动画反馈 | 可与阶段 1 后期并行（当前已启动） |
| 阶段 4（玩法） | 多刀切割 + 风险系统 + 成就框架 | 强依赖前三阶段 |
| 阶段 5（内容） | 数据表扩展 + 故事与图鉴 | 可从阶段 1 后期开始准备 |

**并行原则**：
1. 视觉基础（阶段 1）优先保证
2. 技术优化（阶段 2）已完成，后续专注视觉与UI并行
3. UI/UX（阶段 3）与视觉后期并行，快速提升操作沉浸感
4. 内容生产（阶段 5）可提前准备数据与美术资源
